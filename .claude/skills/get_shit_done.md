# GSD (Get Shit Done) — Правила работы Claude Code

## Основные принципы
1. Всегда выполняй только одну задачу за раз.
2. Не изменяй файлы, которые не относятся к текущей задаче.
3. Перед изменением покажи план (названия файлов и суть изменений).
4. Никогда не начинай новую задачу без явного согласия пользователя.

## После завершения задачи
1. Покажи список изменённых файлов (полные пути).
2. Кратко объясни, что конкретно сделано по каждому файлу.
3. Проверь, что код валиден (нет синтаксических ошибок, закрыты теги).
4. Сообщи: «Задача завершена. Ожидаю следующей команды.»

## Если задача большая (оценка >5 изменяемых файлов или >200 строк кода)
- Разбей её на подзадачи (например, 1. Верстка шапки, 2. Герой, 3. Преимущества...)
- Выполняй подзадачи последовательно, после каждой запрашивая подтверждение.

## Дополнительные экономии токенов
- Не выводи полный код файла, если не просят — только дифф или описание изменений.
- Не генерируй пояснения в стиле «Вот как это работает» — только факт выполнения.
- Не предлагай улучшений, не связанных с текущей задачей.

## Обработка ошибок
- Если команда не ясна — задай один уточняющий вопрос.
- Если нужно установить пакет (npm, pip) — спроси разрешения.
- Никогда не выполняй `rm -rf`, `git push --force` без явного одобрения.

## Критический запрет
- **Не предлагай использовать калькулятор на сайте.** Напоминание: клиент отказался.
- **Не создавай формы отправки на почту без запроса.** Только ссылки на мессенджеры.

## Перед каждым деплоем (git push)
- Запустить `npm run build` (сборка CSS)
- Запустить `npm run lint` (проверка)
- Если ошибок нет: `git add .`, `git commit -m "..."`, `git push`
- Не пушить без успешного выполнения build и lint.
